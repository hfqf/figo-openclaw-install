# OpenClaw 常见使用场景与最佳实践 💡

OpenClaw 是一个强大的 AI Agent 平台，它不仅能陪你聊天，还能帮你干活。以下是几个经过验证的、能显著提升效率的使用场景。

---

## 场景一：全能私人助理 (Personal Assistant)

这是最基础也最常用的场景。你可以把它连接到飞书或 Slack，随时随地呼唤它。

*   **难度**：⭐ (入门)
*   **适用人群**：所有人
*   **能做什么？**
    *   **知识问答**：遇到不懂的技术问题、名词解释，直接问它。
    *   **日程管理**：结合 Calendar 工具，帮你查询和安排日程。
    *   **信息检索**：配置联网搜索工具后，它可以帮你查最新的新闻、股价或技术文档。
*   **配置建议**：
    *   **模型**：推荐 `GPT-5.2` 或 `Claude 4.5 Sonnet`，逻辑与对话能力最强。
    *   **记忆**：开启本地记忆（Memory），它能记住你的喜好和之前的对话。

### 💬 试试这样问 (Prompt Example)：
> "帮我查一下 OpenClaw 最新的版本是多少？如果不确定，请使用 Google 搜索工具确认。"
> "把下周三下午 2 点的‘产品评审会’加到我的日历里。"

---

## 场景二：自动化运维 (DevOps Bot)

把 OpenClaw 变成运维团队的一员，7x24 小时待命。

*   **难度**：⭐⭐⭐ (进阶)
*   **前提条件**：需配置 SSH 或 Kubernetes 工具权限。
*   **能做什么？**
    *   **故障排查**：当报警发生时，让它去查询服务器日志、检查 CPU/内存负载，并给出初步诊断。
    *   **执行脚本**：通过 `Execute Command` 工具，让它安全地执行预定义的运维脚本（如重启服务、清理缓存）。
    *   **状态巡检**：每天定时检查各个服务的健康状态，并生成日报发到群里。
*   **配置建议**：
    *   **安全第一**：务必限制它能执行的命令范围，建议在沙箱（Sandbox）中运行。
    *   **模型**：`GPT-5.2` (推理能力强) 或 `DeepSeek V4` (性价比高)。

### 💬 试试这样问 (Prompt Example)：
> "检查一下服务器 `prod-web-01` 的 CPU 使用率，如果超过 80%，列出占用最高的 5 个进程。"
> "帮我重启一下 `nginx` 服务，并确认重启后状态是 `active`。"

## 场景三：代码助手与结对编程 (Coding Partner)

不仅仅是生成代码，而是参与到你的开发流程中。

*   **难度**：⭐⭐ (中级)
*   **前提条件**：最好本地运行或配置代码仓库访问权限。
*   **能做什么？**
    *   **代码审查 (Code Review)**：把代码片段发给它，让它检查潜在的 Bug 或优化点。
    *   **生成单元测试**：给它一段业务代码，让它自动生成对应的测试用例。
    *   **文档生成**：读代码，自动生成 API 文档或注释。
*   **配置建议**：
    *   **模型**：强烈推荐 `Claude 4.5 Sonnet` 或 `DeepSeek V4`，编程能力顶尖。
    *   **隐私**：如果你担心代码泄露，可以使用本地模型 `Llama 4` 或 `Qwen 2.5 Coder`。

### 💬 试试这样问 (Prompt Example)：
> "这段 Python 代码的性能有优化空间吗？请给出具体的优化建议和修改后的代码。"
> "为这个 `UserLogin` 函数编写一个 Jest 单元测试，覆盖正常登录和密码错误的场景。"

## 场景四：企业知识库问答 (Enterprise Knowledge Base)

利用 OpenClaw 的 RAG（检索增强生成）能力，让它成为企业的“百事通”。

*   **难度**：⭐⭐ (中级)
*   **前提条件**：需要准备好知识库文档 (PDF/Markdown)。
*   **能做什么？**
    *   **员工入职指引**：新员工问“怎么申请报销？”，它直接从公司手册里找答案。
    *   **产品技术支持**：客服人员问“报错代码 1001 是什么意思？”，它从技术文档里检索解决方案。
*   **配置建议**：
    *   **知识导入**：将公司的文档放入 OpenClaw 的 `knowledge` 目录。
    *   **模型**：配置一个轻量级的 Embedding 模型（如 `text-embedding-3-small`）以提高检索速度。

### 💬 试试这样问 (Prompt Example)：
> "根据公司的《财务报销制度》，出差餐饮费的每日上限是多少？"
> "客户遇到了 `Error 500`，请在技术文档中查找可能的原因和解决方案。"

## 场景五：智能数据分析 (Data Analyst)

让 OpenClaw 帮你处理繁琐的 Excel 或 CSV 数据。

*   **难度**：⭐⭐⭐ (进阶)
*   **前提条件**：需开启代码执行环境 (Code Interpreter)。
*   **能做什么？**
    *   **报表生成**：上传一个 CSV 文件，让它帮你统计销售额、计算增长率。
    *   **图表绘制**：让它用 Python (Pandas/Matplotlib) 生成数据可视化图表。
*   **配置建议**：
    *   **环境**：必须开启代码执行（Code Execution）功能，通常基于 Docker。
    *   **模型**：`GPT-5.2-Thinking` 或 `Claude 4.5 Sonnet` (数据敏感度高)。

### 💬 试试这样问 (Prompt Example)：
> "分析这个 CSV 文件中的销售数据，按月份统计总销售额，并画一个柱状图。"
> "找出数据中增长率最高的产品类别，并计算其同比增长率。"

## 💡 进阶技巧：如何写出好的 Prompt（提示词）？

不管在哪个场景，跟 AI 说话都有技巧 (遵循 `CRIS` 原则)：

1.  **C (Context) 背景**：“你是一个资深的 Python 工程师...”
2.  **R (Request) 任务**：“请帮我写一个脚本...”
3.  **I (Instruction) 约束**：“不要使用第三方库，代码要简洁...”
4.  **S (Style) 风格**：“请用幽默的口吻回复...”

**示例**：
> "你是一个资深的数据分析师 (Context)。请帮我分析这份财报 (Request)。重点关注净利润的变化，如果下降了，请用红字标出 (Instruction)。最后生成一个简短的总结报告 (Style)。"
