# OpenClaw 极简维护手册 (小白专用版)

这份手册专为不懂技术、不想折腾代码的用户准备。只要按照这里的说明操作，你就能轻松管理你的 OpenClaw。

---

## 1. 怎么打开 OpenClaw？

通常在安装完成后，你的桌面上会有一个名为 `Start OpenClaw` 的快捷方式（或者脚本）。

*   **启动方法**：双击桌面的启动图标。
    *   *注意*：启动后会自动弹出一个黑框框（命令行窗口），**不要关闭它**，它会自动运行并在完成后打开浏览器。
*   **访问地址**：如果浏览器没有自动打开，请手动在浏览器地址栏输入：
    *   本机访问：`http://localhost` 或 `http://127.0.0.1`
    *   局域网访问：请询问安装人员你的电脑 IP 地址（例如 `http://192.168.1.100`）。

## 2. 飞书机器人不回话怎么办？

如果你的飞书机器人突然“哑巴”了，请按以下顺序检查：

1.  **检查电脑是否开机**：OpenClaw 是运行在你电脑上的，电脑关机或休眠了，机器人就下班了。
2.  **检查 OpenClaw 是否运行**：
    *   打开浏览器访问一下管理后台，如果网页打不开，说明服务挂了。请重新双击桌面的启动图标。
3.  **检查网络**：
    *   如果你用了海外模型（如 ChatGPT），请确认你的魔法（代理软件）是否开启，并且节点是通畅的。

## 3. 怎么重启服务？

如果遇到莫名其妙的卡顿或错误，重启通常能解决 90% 的问题。

1.  找到之前启动的那个黑框框，关闭它。
2.  等待几秒钟。
3.  重新双击桌面的启动图标。

## 4. 数据备份（防丢必看）

虽然 OpenClaw 很稳定，但为了防止意外，建议定期备份。

*   **你需要备份什么？**
    *   找到安装目录下的配置文件（如 `.env`）。
    *   找到安装目录下名为 `data` 或 `volumes` 的文件夹（如果有）。
*   **怎么备份？**
    *   直接把整个 OpenClaw 安装文件夹复制一份，粘贴到你的移动硬盘或网盘里。搞定！

## 5. 常见报错“人话”翻译

| 报错关键词 | 意思 | 怎么办 |
| :--- | :--- | :--- |
| `Connection refused` | 连不上网 | 检查代理软件，或者检查 OpenClaw 是否启动。 |
| `502 Bad Gateway` | 服务正在启动中 | 别急，让它再跑一会儿（通常需要 1-2 分钟）。 |
| `EADDRINUSE` | 端口被占用 | 你的电脑上有其他软件占用了 OpenClaw 需要的端口。请尝试重启电脑，或关闭其他开发工具。 |
| `spawn EINVAL` | Windows 抽风了 | 这是一个 Windows 特有的命令执行错误。通常重启服务（关闭黑框框再打开）就能解决。如果一直报错，请联系技术人员重新安装插件。 |
| `model not allowed` | 模型没权限 | 你想用的 AI 模型没在白名单里。请联系管理员在后台添加该模型。 |
| `Unauthorized` | 密码不对 | 你的登录凭证过期了。请尝试重新登录 Dashboard，或者重新扫码绑定。 |
| `Network Error` | 网断了 | 检查你的网络连接，或者检查代理软件是否正常工作（特别是用 ChatGPT 时）。 |

## 6. 进阶：自己当医生 (简单的自我诊断)

如果你稍微懂一点点电脑操作，可以试着运行这几个“神医”命令。

1.  **打开命令行**：在 OpenClaw 安装目录下，右键选择“在终端中打开”或“Open in Terminal”。
2. **输入命令**：
    *   查看服务状态（使用 openclaw status 或类似命令）。

3. **查看日志**：
   - 打开终端，运行 `openclaw logs`（或根据实际启动方式查看日志）。
   - 看看有没有红色的报错信息。
