## 🚀 1. 快速启动 (Quick Start)

通常在安装完成后，你的桌面上会有一个名为 `Start OpenClaw` 的快捷方式（或者脚本）。

### 启动步骤：
1.  **双击图标**：双击桌面的 `Start OpenClaw` 启动图标。
2.  **保持运行**：启动后会自动弹出一个黑框框（命令行窗口），**千万不要关闭它**！它会自动运行并在完成后打开浏览器。
3.  **访问后台**：
    *   如果浏览器没有自动打开，请手动在浏览器地址栏输入：
    *   **本机访问**：[http://localhost](http://localhost) 或 [http://127.0.0.1](http://127.0.0.1)
    *   **局域网访问**：请询问安装人员你的电脑 IP 地址（例如 `http://192.168.1.100`）。

## ❓ 3. 常见问题急救包

### Q1: 飞书/Slack 机器人突然不回话了？
请按以下顺序检查：
1.  **检查电脑状态**：OpenClaw 是运行在你电脑上的，电脑关机、休眠或锁屏导致断网，机器人都会“下班”。
2.  **检查服务状态**：打开浏览器访问管理后台，如果网页打不开，说明服务挂了。请重新双击桌面的启动图标。
3.  **检查网络连接**：
    *   如果你用了海外模型（如 GPT-5.2），请确认你的网络环境（代理/魔法）是否通畅。
    *   尝试在浏览器打开 `google.com` 确认网络是否正常。

### Q2: 怎么重启服务？
遇到卡顿、报错或莫名其妙的问题，**重启**能解决 90% 的烦恼：
1.  找到之前启动的那个黑框框（终端窗口）。
2.  点击右上角的 `X` 关闭它。
3.  等待 5 秒钟。
4.  重新双击桌面的 `Start OpenClaw` 图标。

### Q3: 报错看不懂怎么办？(报错翻译机)

| 报错关键词 | 意思 | 怎么办 |
| :--- | :--- | :--- |
| `Connection refused` | 连不上网/服务未启动 | 检查 OpenClaw 是否启动；检查代理软件。 |
| `502 Bad Gateway` | 服务正在启动中 | 别急，让它再跑一会儿（通常需要 1-2 分钟）。 |
| `EADDRINUSE` | 端口被占用 | 有其他软件占用了端口。**重启电脑**通常能解决。 |
| `spawn EINVAL` | Windows 系统抽风 | 常见于 Windows。关闭黑框框，**重新启动**即可。 |
| `model not allowed` | 模型没权限 | 你想用的 AI 模型没在白名单里。请联系管理员添加。 |
| `Unauthorized` | 密码/Token 过期 | 重新登录后台，或重新配置飞书/Slack 的凭证。 |
| `Quota exceeded` | 额度用完了 | 你的 API Key 没钱了，请充值或更换 Key。 |

## 🛡️ 4. 数据备份（防丢必看）

虽然 OpenClaw 很稳定，但为了防止意外（如电脑重装），建议定期备份。

*   **你需要备份什么？**
    1.  **配置文件**：安装目录下的 `.env` 文件（存储了密钥和配置）。
    2.  **数据目录**：安装目录下的 `data/` 或 `volumes/` 文件夹（如果有，存储了对话记录）。
*   **怎么备份？**
    *   最简单的方法：直接把**整个 OpenClaw 安装文件夹**复制一份，粘贴到你的移动硬盘或网盘里。搞定！

## 🛠️ 5. 进阶：简单的自我诊断

如果你稍微懂一点点电脑，可以试着运行这几个“神医”命令：

1.  **打开命令行**：在 OpenClaw 安装目录下，右键空白处，选择“在终端中打开” (Open in Terminal)。
2.  **查看日志**：
    *   输入 `openclaw logs` (或者 `docker logs openclaw` 如果是 Docker 部署)。
    *   看看有没有红色的 `Error` 信息，截图发给技术支持。
